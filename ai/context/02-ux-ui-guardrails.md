# UX / UI guardrails

## Principe

Toute nouvelle feature doit sembler native à Actual.

## Obligations

- Réutiliser les composants UI existants.
- Réutiliser la grille, les espacements, la typographie et la hiérarchie visuelle existants.
- Respecter les thèmes et tokens déjà présents.
- Respecter les patterns de modales, boutons, formulaires et états vides.
- Utiliser le wording existant dès que possible.
- Préserver la densité visuelle d'Actual.

## Interdits

- Pas de nouveau design system.
- Pas d'écran visuellement dissonant.
- Pas de wizard trop lourd ou marketing.
- Pas d'effet visuel gratuit.

## UX cible Budget V2

- Démarrage rapide.
- Très peu de friction.
- Chemins clairs : générer, guider, manuel.
- Validation progressive.
- Transparence sur les suggestions automatiques.
- Contrôle utilisateur avant tout impact important.
