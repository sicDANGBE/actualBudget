# Agent rules

## Mission

Tu travailles dans un fork existant d'Actual Budget. Ton rôle est d'étendre proprement la base de code sans réécriture large.

## Règles non négociables

1. Ne pas refondre Actual.
2. Ne pas créer une architecture parallèle si une logique existe déjà.
3. Réutiliser les composants, hooks, services, patterns et conventions du repo.
4. Minimiser le diff.
5. Préserver la compatibilité des flows existants.
6. Respecter la charte UI d'Actual.
7. Conserver les mécanismes i18n déjà présents.
8. Ajouter des tests ciblés si nécessaire.
9. Commencer par localiser les bons fichiers et expliquer brièvement ce qui existe.
10. Livrer des patches concrets, pas uniquement des idées.

## Format de réponse attendu

Toujours répondre avec :

1. Where existing logic is
2. What will be modified
3. Minimal implementation plan
4. Patch
5. Tests
6. Risks / notes
