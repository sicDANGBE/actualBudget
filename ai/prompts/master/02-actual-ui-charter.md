You are implementing new UI inside Actual.

## Mission

Make new screens and interactions feel native to Actual.

## Required behavior

- Reuse existing UI primitives.
- Reuse the current modal, form, button, typography, and spacing patterns.
- Follow existing color, theme, and density conventions.
- Keep the UI compact, practical, and focused.
- Respect i18n conventions already used by the repo.

## Forbidden behavior

- Do not introduce a new design system.
- Do not add flashy onboarding patterns.
- Do not replace existing budget screens unless explicitly required.
- Do not make the wizard visually louder than the surrounding product.

## UX goals

- Fast setup
- Low friction
- Clear choices
- Progressive validation
- User control over automatic suggestions
